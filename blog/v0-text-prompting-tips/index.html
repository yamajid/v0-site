Prompting Fundamentals

Effective prompting is the key to getting great results from v0. The AI responds best to clear, structured, and specific instructions.

Core principles:

  • Be explicit: Don't assume v0 knows your preferences
  • Use lists: Break down complex requirements
  • Provide context: Explain the use case when helpful
  • Reference examples: Mention similar products or patterns
  • Iterate: Refine with follow-up prompts

The Golden Rule

Write prompts like you're briefing a talented developer who doesn't know your project. Give enough detail to build it right the first time.

Prompt Structure

A well-structured prompt has distinct parts:

1. Component Type

Start with what you're building:

"Create a [component type]..."
"Build a [page/section]..."
"Design a [feature]..."

2. Core Requirements

List the essential elements:

"...with:
- Element 1
- Element 2
- Element 3"

3. Behavior/Interactions

Describe how it should work:

"When the user clicks X, Y should happen"
"On hover, show Z"
"Support keyboard navigation"

4. Style Direction (Optional)

Guide the visual approach:

"Use a minimalist design"
"Match Apple's aesthetic"
"Dark theme with purple accents"

Complete Example

"Create a testimonial carousel with:
- 3 testimonial cards showing quote, author name, 
  title, and company
- Auto-play that pauses on hover
- Dot indicators for current slide
- Previous/next arrow buttons
- Smooth sliding animation
- Responsive: 1 card on mobile, 3 on desktop

Use a clean, modern design with subtle shadows 
and rounded corners."

Being Specific

Vague vs. Specific

Bad: "Make a card component"

Good: "Create a product card with image (16:9), title, price, rating stars, and 'Add to Cart' button. Include hover effect that lifts the card. Price should show original price struck through if on sale."

Specify Numbers

  • Instead of "some items" → "5 items"
  • Instead of "padding" → "24px padding"
  • Instead of "columns" → "3 columns on desktop, 1 on mobile"

Name Interactions

// Be explicit about states
"Button states: 
- Default: filled blue
- Hover: darker blue with slight scale
- Active: even darker
- Disabled: gray, reduced opacity, no hover effect
- Loading: show spinner, disable clicks"

Describe Layout Precisely

"Layout: 
- Sidebar fixed at 280px width on left
- Main content fills remaining space
- Header 64px height, sticky
- Content area has 32px padding
- Footer at bottom, not fixed"

Iterative Refinement

Complex components are built through conversation, not single prompts.

Start Broad, Then Refine

  1. Initial prompt: "Create a dashboard layout with sidebar, header, and main content"
  2. Add navigation: "Add navigation items to the sidebar: Dashboard, Projects, Team, Settings"
  3. Add header content: "Put a search bar and user avatar dropdown in the header"
  4. Style adjustments: "Make the sidebar darker and add icons to each nav item"

Refinement Prompt Patterns

// Adding features
"Add a [feature] to the [component]"

// Modifying style
"Make the [element] [style change]"

// Changing behavior
"When [trigger], [new behavior]"

// Fixing issues
"The [element] should [correct behavior] instead"

// Requesting variations
"Show me 3 variations of the [element]"

Don't Start Over

If v0 generated something close but not perfect, refine rather than restart:

Bad: Rewriting entire prompt from scratch

Good: "Keep the layout but change the color scheme to dark mode"

Common Prompt Patterns

Pattern: Reference Comparison

"Create a notification dropdown similar to 
GitHub's, with:
- Unread indicator badge
- List of notifications grouped by repo
- Mark all as read button
- Link to notification settings"

Pattern: User Story Format

"As a user, I want to filter products so I can 
find what I'm looking for quickly.

Create a filter sidebar with:
- Price range slider
- Category checkboxes
- Brand multi-select
- Rating minimum selector
- 'Clear all' and 'Apply' buttons"

Pattern: State Enumeration

"Create a file upload component with states:
- Empty: dashed border, 'Drag files here' text
- Drag over: highlighted border, 'Drop to upload'
- Uploading: progress bar with percentage
- Success: green check, file name, remove button
- Error: red border, error message, retry button"

Pattern: Responsive Specification

"Create a pricing section:
- Desktop (1024px+): 3 cards side by side
- Tablet (768-1023px): 3 cards, tighter spacing
- Mobile (<768px): cards stacked vertically

Middle card should be highlighted as 'Popular' 
with a badge and slight elevation."

Pattern: Technology Request

"Build a data table using @tanstack/react-table with:
- Column sorting
- Global search filter
- Pagination (10/25/50 per page)
- Row selection with checkbox column
- Bulk action toolbar when rows selected"

Mistakes to Avoid

1. Being Too Vague

"Make it look nice"

"Add some animations"

"Make it responsive"

These leave too much to interpretation. Be specific about what "nice," "animations," or "responsive" mean to you.

2. Overloading Single Prompts

500-word prompt describing an entire application

Break complex requirements into multiple focused prompts.

3. Assuming Context

"Add the usual buttons"

"Use our standard colors"

v0 doesn't know your standards. Specify exactly what you want.

4. Ignoring States

Only describing the happy path

Always specify loading, error, and empty states for data-driven components.

5. Forgetting Mobile

Describing desktop layout only

Include mobile considerations or explicitly request responsive design.

Prompt Library

Navigation

"Create a responsive navbar with:
- Logo on left
- Navigation links in center (Desktop only)
- Search icon, notification bell, user avatar on right
- Mobile: hamburger menu opening slide-out drawer
- Sticky on scroll with backdrop blur"

Hero Section

"Build a SaaS hero section with:
- Large headline (max 6 words)
- Supporting paragraph (2-3 lines)
- Email capture form with 'Get Started' button
- Trust indicators: '10,000+ teams' with avatars
- Abstract gradient background, no images"

Feature Grid

"Create a features section with 6 feature cards:
- Icon, title, description per card
- 3 columns on desktop, 2 on tablet, 1 on mobile
- Subtle hover effect (lift + shadow)
- Each card links somewhere (use # for now)"

Pricing Table

"Design a pricing section with 3 tiers:
- Free ($0), Pro ($29/mo), Enterprise (Contact)
- List of features with check/x marks
- Pro tier highlighted as 'Most Popular'
- Annual toggle showing 20% discount
- CTA buttons: 'Get Started' / 'Start Trial' / 'Contact Sales'"

Dashboard Card

"Create a metrics card showing:
- Label, value, and % change from last period
- Positive change = green arrow up
- Negative change = red arrow down
- Small sparkline chart
- Click expands to detailed view"

Conclusion

Effective prompting is a skill that improves with practice. Start with clear, structured prompts that specify the component, its elements, behaviors, and style. Iterate with focused follow-ups rather than rewriting from scratch.

Remember: v0 is powerful but not psychic. The more clearly you communicate your vision, the better it can build it for you. Save prompts that work well, build your own library, and refine your prompting style over time.
