What is Full-Stack v0?

v0 has evolved from a UI component generator to a complete full-stack development platform. You can now build entire applications with frontend, backend, database, and authentication—all through natural language prompts and AI assistance.

Full-stack v0 generates Next.js applications using the App Router, which provides:

  • Server Components: Components that render on the server for better performance
  • Server Actions: Backend functions called directly from components
  • API Routes: REST endpoints for external integrations
  • Database connections: Direct integration with Postgres, Supabase, etc.

What You Can Build

  • SaaS applications with user accounts
  • E-commerce stores with payments
  • Content management systems
  • Internal business tools
  • Real-time collaborative apps

Application Architecture

v0 generates applications following Next.js best practices:

Project Structure

my-app/
├── app/
│   ├── layout.tsx        # Root layout
│   ├── page.tsx          # Homepage
│   ├── api/              # API routes
│   │   └── users/
│   │       └── route.ts  # REST endpoint
│   ├── dashboard/
│   │   └── page.tsx      # Dashboard page
│   └── actions/          # Server actions
│       └── user-actions.ts
├── components/
│   └── ui/               # UI components
├── lib/
│   ├── db.ts             # Database client
│   └── auth.ts           # Auth helpers
└── types/
    └── index.ts          # TypeScript types

Data Flow

  1. User interacts with UI component
  2. Server Action or API route is called
  3. Backend logic validates and processes
  4. Database is queried or updated
  5. Response flows back to UI
  6. UI updates with new data

Server Actions

Server Actions are the modern way to handle form submissions and mutations in Next.js. v0 generates them automatically.

Creating a Server Action

Prompt v0:

"Create a contact form that saves submissions 
to the database with name, email, and message"

v0 generates:

// app/actions/contact-actions.ts
'use server'

import { db } from '@/lib/db'
import { z } from 'zod'
import { revalidatePath } from 'next/cache'

const contactSchema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  message: z.string().min(10)
})

export async function submitContact(formData: FormData) {
  const validated = contactSchema.parse({
    name: formData.get('name'),
    email: formData.get('email'),
    message: formData.get('message')
  })

  await db.contact.create({
    data: validated
  })

  revalidatePath('/contact')
  return { success: true }
}

Using Server Actions in Components

// components/contact-form.tsx
import { submitContact } from '@/app/actions/contact-actions'

export function ContactForm() {
  return (
    <form action={submitContact}>
      <input name="name" placeholder="Your name" />
      <input name="email" type="email" placeholder="Email" />
      <textarea name="message" placeholder="Message" />
      <button type="submit">Send</button>
    </form>
  )
}

Server Action Benefits

  • No separate API endpoint needed
  • Type-safe from frontend to backend
  • Automatic form handling
  • Works without JavaScript (progressive enhancement)

API Routes

For REST APIs or webhooks, v0 generates API routes:

Creating an API Route

Prompt:

"Create a REST API for products with GET all, 
GET by id, POST create, PUT update, DELETE"

v0 generates:

// app/api/products/route.ts
import { NextResponse } from 'next/server'
import { db } from '@/lib/db'

export async function GET() {
  const products = await db.product.findMany()
  return NextResponse.json(products)
}

export async function POST(request: Request) {
  const body = await request.json()
  const product = await db.product.create({ data: body })
  return NextResponse.json(product, { status: 201 })
}

// app/api/products/[id]/route.ts
export async function GET(
  request: Request,
  { params }: { params: { id: string } }
) {
  const product = await db.product.findUnique({
    where: { id: params.id }
  })
  if (!product) {
    return NextResponse.json(
      { error: 'Not found' },
      { status: 404 }
    )
  }
  return NextResponse.json(product)
}

Database Integration

v0 supports multiple database providers and generates appropriate code for each.

Setting Up Database

  1. In v0 settings, connect your database (Vercel Postgres, Supabase, etc.)
  2. Describe your data model in prompts
  3. v0 generates schema and queries

Example: E-commerce Schema

Prompt:

"Set up database schema for an e-commerce store 
with products (name, price, description, stock), 
categories, and orders with line items"

v0 generates Drizzle or Prisma schema and all necessary queries.

Query Patterns

// Fetch with relations
const ordersWithItems = await db.order.findMany({
  include: {
    items: {
      include: { product: true }
    },
    user: true
  }
})

// Aggregations
const salesByCategory = await db.orderItem.groupBy({
  by: ['categoryId'],
  _sum: { total: true }
})

Authentication

v0 can generate authentication flows using NextAuth.js or Clerk.

Adding Auth with NextAuth

Prompt:

"Add authentication with email/password and 
Google OAuth. Include sign up, sign in, and 
password reset flows"

v0 generates:

  • NextAuth configuration
  • Sign in/sign up pages
  • Protected route middleware
  • Session handling hooks

Protecting Routes

// middleware.ts
import { withAuth } from 'next-auth/middleware'

export default withAuth({
  pages: {
    signIn: '/auth/signin'
  }
})

export const config = {
  matcher: ['/dashboard/:path*', '/settings/:path*']
}

Using Session in Components

import { getServerSession } from 'next-auth'
import { authOptions } from '@/lib/auth'

export default async function DashboardPage() {
  const session = await getServerSession(authOptions)
  
  if (!session) {
    redirect('/auth/signin')
  }
  
  return <div>Welcome, {session.user.name}</div>
}

Complete Example: Task Manager

Let's walk through building a complete task management app with v0.

Step 1: Describe the Application

"Build a task management app where users can:
- Sign up and log in
- Create projects
- Add tasks to projects with due dates and priority
- Mark tasks complete
- Filter and sort tasks
Use Vercel Postgres for the database"

Step 2: v0 Generates the Foundation

v0 creates:

  • Database schema for users, projects, tasks
  • Authentication setup
  • Dashboard layout
  • Project and task CRUD operations

Step 3: Refine with Follow-ups

"Add drag-and-drop to reorder tasks"
"Make the task list update in real-time"
"Add a calendar view for tasks with due dates"

Step 4: Polish the UI

Use Design Mode to adjust colors, spacing, and animations without prompts.

Step 5: Deploy

One-click deploy to Vercel with automatic database connection.

Full-Stack Prompt Tips

  • Start with data model: Describe entities and relationships first
  • Specify auth needs: What users can do, role-based access
  • Include validation: "Validate email format and require password strength"
  • Think about errors: "Handle network errors gracefully with retry"
